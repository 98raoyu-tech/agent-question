## 标准答案

Transformer 由 Vaswani 等人于 2017 年在论文《Attention Is All You Need》中提出，主要解决了以下核心问题：

1. **序列建模的长距离依赖问题**：传统 RNN/LSTM 在处理长序列时，信息必须逐步传递，导致远距离信息衰减严重。Transformer 通过自注意力机制实现任意两个位置之间的直接连接，路径长度为 O(1)。

2. **计算并行化问题**：RNN 必须按时间步顺序计算（O(n) 序列长度步），无法充分利用 GPU 并行能力。Transformer 的自注意力可以对整个序列并行计算，训练效率大幅提升。

3. **信息瓶颈问题**：Encoder-Decoder 架构中的传统方法依赖固定长度的上下文向量（context vector），而 Transformer 使用交叉注意力动态关注源序列的不同部分，消除了这一瓶颈。

4. **泛化能力与统一架构**：同一架构可迁移至 NLP、CV、语音、多模态等多种任务，奠定了"基础模型（Foundation Model）"的技术基础。

## 深度解析

**为什么 RNN 有长距离依赖问题？**

RNN 的隐藏状态 h_t = f(h_{t-1}, x_t)，这意味着第 t 步的信息需要经过 t 次非线性变换才能到达第 0 步。反向传播中梯度需要连乘 t 次权重矩阵，导致梯度消失或爆炸。LSTM/GRU 通过门控机制缓解了这一问题，但并未从根本上解决——信息仍需顺序传递。

**Transformer 如何解决？**

自注意力计算中，Query 和 Key 的点积可以直接衡量任意两个位置的相关性，Value 负责聚合信息。这相当于将信息传递路径从 O(n) 缩短到 O(1)——每个位置都可以直接"看到"其他所有位置。

**并行化的本质**

Transformer 摒弃了递归结构，改用矩阵乘法完成序列处理。对于长度为 n、维度为 d 的序列，自注意力的核心操作是 Q·K^T 和 softmax(Q·K^T)·V，均为可并行的矩阵运算。这使得 Transformer 在现代 GPU/TPU 上的吞吐量远超 RNN。

## 🎯 面试追问

**Q1：Transformer 能完全替代 RNN 吗？在什么场景下 RNN 仍然有优势？**

Transformer 在绝大多数 NLP 任务上已全面超越 RNN。但在以下场景 RNN 仍可能有优势：
- 极长序列的在线/流式处理（Mamba 等 SSM 架构正在弥合这一差距）
- 极低算力的边缘设备部署（RNN 参数量和计算量更小）
- 需要严格因果顺序的时序信号处理

**Q2：Transformer 的自注意力计算复杂度是多少？这带来了什么限制？**

自注意力的时间和空间复杂度均为 O(n²·d)，其中 n 为序列长度，d 为维度。这意味着当序列长度增加时，计算量和显存占用呈平方增长。这也是为什么原始 Transformer 限制在 512/1024 token，后来催生了 Flash Attention、Sparse Attention、线性注意力等优化方案。

**Q3：Transformer 最初是为 NLP 设计的，为什么能迁移到 CV 和其他领域？**

因为 Transformer 的核心计算——自注意力机制——是 domain-agnostic 的。它只关心输入元素之间的相关性，不依赖于序列的线性结构。ViT 将图像切分为 patch 序列，Audio Transformer 将音频切分为帧序列，都是将领域数据转化为"token 序列"后复用 Transformer 架构。

## 🚀 AI应用扩展

在企业级 AI 系统中，Transformer 架构的选型直接影响系统性能和成本：

- **RAG 系统**：Embedding 模型基于 Transformer Encoder（如 BGE、E5），用于将文档和查询编码为向量。Transformer 的并行性使得大规模文档索引成为可能。
- **智能客服**：基于 Transformer Decoder（如 GPT-4、Qwen）的对话模型，自注意力机制使模型能同时关注对话历史中的多个关键信息。
- **代码生成**：CodeLlama、DeepSeek-Coder 等模型利用 Transformer 的长距离依赖能力，理解复杂的代码结构和跨文件引用。
- **多模态应用**：CLIP、GPT-4V 等模型证明 Transformer 可以统一处理文本、图像、音频等模态，为企业提供统一的 AI 基础设施。

## ⚠️ 容易踩坑

1. **误以为 Transformer 完全没有序列限制**：虽然 Transformer 没有 RNN 的逐步传递问题，但其 O(n²) 复杂度和固定位置编码长度仍然限制了有效上下文长度。
2. **混淆 Transformer 架构与具体模型**：Transformer 是一种架构范式，GPT/BERT/T5 是基于 Transformer 的具体模型实现，它们的结构差异很大。
3. **忽略位置信息的重要性**：Transformer 的自注意力本身是位置无关的（置换不变性），必须通过位置编码注入序列顺序信息。

## 标准答案

自注意力机制（Self-Attention）是 Transformer 的核心计算单元，它允许序列中的每个位置都能直接关注并聚合序列中所有其他位置的信息。

**核心计算过程：**

给定输入序列 X ∈ ℝ^{n×d}，通过三个线性变换得到：
- **Query（查询矩阵）** Q = XW_Q，W_Q ∈ ℝ^{d×d_k}
- **Key（键矩阵）** K = XW_K，W_K ∈ ℝ^{d×d_k}
- **Value（值矩阵）** V = XW_V，W_V ∈ ℝ^{d×d_v}

注意力计算公式：

```
Attention(Q, K, V) = softmax(QK^T / √d_k) · V
```

其中 √d_k 是缩放因子，用于防止点积值过大导致 softmax 梯度消失。

**直觉理解**：每个 token 生成一个 Query 向量表示"我在找什么"，每个 token 生成一个 Key 向量表示"我有什么"，Q 和 K 的点积衡量匹配程度，最终用匹配分数对 Value 加权求和，得到融合了全局信息的新表示。

## 深度解析

**为什么要用 Q、K、V 三个矩阵？**

直接用 X 与 X 做点积也可以计算相似度，但这样只有一个"视角"。Q、K、V 的分离提供了三个独立的语义空间：
- Q 空间：编码"需要什么样的信息"
- K 空间：编码"我能提供什么信息"
- V 空间：编码"实际传递的内容"

这种分离使得模型可以在不同的语义空间中进行匹配和聚合，大幅提升了表达能力。

**为什么要除以 √d_k？**

假设 Q 和 K 的每个元素独立同分布，均值为 0，方差为 1，则 Q·K^T 的每个元素的方差为 d_k。当 d_k 很大时，点积值会很大，softmax 会趋近于 one-hot 分布（梯度趋近于 0）。除以 √d_k 将方差归一化为 1，使 softmax 保持在梯度友好的范围内。

**Softmax 的作用**

softmax 将原始分数转换为概率分布（所有权重之和为 1），这保证了加权求和的数值稳定性，同时引入了非线性，使模型可以学习"关注谁"。

## 🏠 生活类比

自注意力就像**图书馆检索系统**：

- 你（Query）带着一个问题走进图书馆："我要找关于深度学习的书"
- 书架上每本书的索引卡（Key）写着它包含的主题
- 你把问题跟每张索引卡匹配，得到相关性分数
- 最相关的书（Value）获得更高的权重
- 最终你综合所有书的内容，得到一个加权后的答案

关键洞察：每本书不需要"排队"等你读完才能推荐下一本——它们同时被检索（并行计算）。

## 🎯 面试追问

**Q1：自注意力的时间复杂度和空间复杂度分别是什么？**

时间复杂度 O(n²·d)，空间复杂度 O(n²)（需要存储 n×n 的注意力矩阵）。其中 n 是序列长度，d 是隐藏维度。Flash Attention 通过分块计算和在线 softmax 将空间复杂度降至 O(n)。

**Q2：为什么 Self-Attention 叫"Self"？它和普通的 Attention 有什么区别？**

"Self" 表示 Q、K、V 来自同一个序列——输入自己关注自己。而普通 Attention（如 Cross-Attention）中，Q 来自一个序列（如 Decoder），K 和 V 来自另一个序列（如 Encoder），用于跨序列的信息聚合。

**Q3：如果不用 Softmax，可以用其他函数替代吗？**

可以。常见替代方案包括：
- **线性注意力**：用 ELU+1 替代 softmax，将复杂度降至 O(n·d²)
- **ReLU 注意力**：用 ReLU 替代 softmax，已在某些场景下验证有效
- **Sigmoid 注意力**：允许每个位置独立地选择关注/不关注，而非竞争式分配

## 🚀 AI应用扩展

- **语义搜索**：Embedding 模型内部的 Self-Attention 帮助模型理解查询和文档的语义，比传统 TF-IDF/BM25 更能捕获语义相似性。
- **智能摘要**：Self-Attention 的注意力权重可以直接用于抽取式摘要——对 [CLS] token 的注意力分布反映了每个句子的重要性。
- **实体关系抽取**：在知识图谱构建中，Self-Attention 可以自动发现实体之间的远程依赖关系。
- **代码审查 AI**：Self-Attention 帮助模型理解代码中变量的引用关系、函数调用链等跨距离依赖。

## ⚠️ 容易踩坑

1. **忘记缩放因子 √d_k**：不除以 √d_k 会导致点积值过大，softmax 输出趋近于 one-hot，梯度几乎为零，模型无法训练。
2. **混淆 Self-Attention 和 Cross-Attention**：Self-Attention 的 Q/K/V 来自同一序列，Cross-Attention 的 Q 来自 Decoder、K/V 来自 Encoder。
3. **误以为注意力权重就是"解释性"**：虽然注意力权重提供了一定的可解释性，但多项研究表明，注意力权重与特征重要性并不总是一致，不能简单等同于"模型在看哪里"。

## 标准答案

多头注意力（Multi-Head Attention, MHA）是 Transformer 中的核心机制，它将自注意力在多个独立的子空间中并行执行，使模型能够同时关注来自不同表示子空间的信息。

**公式定义：**

```
MultiHead(Q, K, V) = Concat(head₁, head₂, ..., headₕ) · W_O
其中 headᵢ = Attention(QWᵢ_Q, KWᵢ_K, VWᵢ_V)
```

- h 为注意力头数（原始 Transformer 使用 h=8）
- 每个头的维度 d_k = d_v = d_model / h
- W_O ∈ ℝ^{d_model × d_model} 为输出投影矩阵

**核心价值**：不同的头可以学习关注不同类型的语言关系——例如一个头关注句法结构（主谓关系），另一个头关注语义相似性（同义词），第三个头关注位置关系（相邻词）。

## 深度解析

**为什么多头比单头好？**

从信息论角度看：
- 单头注意力计算一个 n×n 的注意力矩阵，只能学习一种关注模式
- 多头注意力计算 h 个 n×n 的注意力矩阵，每个矩阵可以学习不同的关注模式
- 最终通过拼接和线性投影，将多种关注模式的信息融合在一起

**参数量分析**

- 单头（维度为 d）：3 个 d×d 矩阵 = 3d² 参数
- h 个头（每个维度 d/h）：h × 3 × (d/h)² = 3d²/h 参数（加上输出投影 d²）
- 总参数量：3d²/h × h + d² = 4d²，与单头注意力的 3d² + d² = 4d² 参数量相同！

也就是说，多头注意力在不增加参数量的情况下，提供了更丰富的表示能力。

**各头学到了什么？**

研究（Clark et al., 2019）发现，BERT 的不同头确实学到了不同的语言知识：
- 某些头关注句法依存关系
- 某些头关注共指消解
- 某些头关注相邻 token
- 某些头类似于 skip-gram（关注特定距离的词）

## 🎯 面试追问

**Q1：多头注意力的头数 h 如何选择？**

原始 Transformer 中 h=8，d_model=512，因此 d_k=64。一般经验：
- d_k 不能太小（至少 32-64），否则表达能力不足
- d_k 不能太大，否则失去了多头的意义
- 实践中 h 通常为 8、12、16、32、64，与 d_model 配合调整
- 有研究表明并不是所有头都有用，可以在训练后进行 Head Pruning

**Q2：MHA、MQA、GQA 分别是什么？有什么区别？**

- **MHA（Multi-Head Attention）**：每个头有独立的 Q、K、V 投影
- **MQA（Multi-Query Attention）**：所有头共享同一组 K、V，只有 Q 是独立的。极大减少 KV Cache 大小，推理速度提升 30-50%
- **GQA（Grouped-Query Attention）**：将 h 个头分为 g 组，每组共享 K、V。是 MHA 和 MQA 的折中方案。Llama 2 70B、Mistral 等模型采用

**Q3：为什么输出要乘一个 W_O？直接 Concat 不行吗？**

直接 Concat 后各头的信息是独立的，没有交互。W_O 的作用是：
1. 将各子空间的信息进行线性融合
2. 将维度映射回 d_model
3. 提供额外的表示能力

- **KV Cache 优化**：在大模型推理中，MHA 的 KV Cache 占用大量显存。GQA 通过分组共享 KV，在保持质量的同时大幅降低显存需求，是 2024-2026 年大模型推理优化的关键技术。

## ⚠️ 容易踩坑

1. **误以为每个头的维度是 d_model**：每个头的维度是 d_model / h，拼接后才恢复为 d_model。

3. **认为头越多越好**：研究表明很多头是冗余的，可以通过剪枝去除。增加头数不一定会提升性能，反而增加计算量。

## 标准答案

位置编码（Positional Encoding）是 Transformer 中用于向输入注入序列位置信息的机制。

**为什么需要？**

自注意力机制具有**置换不变性（Permutation Invariance）**——打乱输入 token 的顺序，自注意力的输出只是相应地重排，注意力权重不变。这意味着 Transformer 无法区分"猫吃鱼"和"鱼吃猫"。因此，必须通过某种方式注入位置信息。

**原始 Transformer 的正弦位置编码：**

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

- pos 是 token 在序列中的位置（0, 1, 2, ...）
- i 是维度索引
- 不同频率的正弦/余弦波编码不同粒度的位置信息

**2026 年主流方案**：
- **RoPE（Rotary Position Embedding）**：在 Q、K 上施加旋转变换，Llama、Qwen、DeepSeek 等主流模型采用
- **ALiBi（Attention with Linear Biases）**：直接在注意力分数上加位置相关的偏置

## 深度解析

**正弦编码的数学直觉**

正弦位置编码有一个关键性质：PE(pos+k) 可以表示为 PE(pos) 的线性函数。这是因为旋转矩阵的性质——对任意固定偏移 k，存在一个线性变换 M_k 使得 PE(pos+k) = M_k · PE(pos)。这意味着模型可以通过线性变换学习相对位置关系。

**绝对位置 vs 相对位置**

- **绝对位置编码**（如正弦编码、可学习编码）：每个位置有固定的编码，与内容无关
- **相对位置编码**（如 RoPE、ALiBi）：编码的是 token 之间的相对距离，更适合泛化到训练时未见过的长度

**RoPE 的核心思想**

RoPE 将位置编码转化为对 Q 和 K 向量的旋转操作。对于位置 m 的 Q 和位置 n 的 K，它们的点积只依赖于相对位置 (m-n) 和原始的 Q·K 值：

```
⟨f_q(x_m, m), f_k(x_n, n)⟩ = g(x_m, x_n, m-n)
```

这使得模型天然具有相对位置感知能力，且无需额外参数。

**ALiBi 的核心思想**

ALiBi 不修改输入嵌入，而是直接在注意力分数上加一个与距离成正比的偏置：

```
Attention = softmax(QK^T / √d_k + m · bias_matrix)
```

其中 bias_matrix[i,j] = -|i-j|，m 是每个头不同的斜率。距离越远，惩罚越大，自然形成局部注意力偏好。

## 🎯 面试追问

**Q1：RoPE 相比正弦位置编码的核心优势是什么？**

1. 天然编码相对位置信息（点积只依赖相对距离）

3. 无需额外参数（正弦编码也不需要，但可学习位置编码需要）
4. 在长序列任务上表现更好

**Q2：ALiBi 和 RoPE 哪个更好？**

- **ALiBi**：实现简单、长度外推能力强，但可能在长距离关系建模上受限（因为距离惩罚会衰减远距离注意力）
- **RoPE**：表达能力更强，是目前主流选择（Llama、Qwen、DeepSeek 等均采用），但长度外推需要额外技术（YaRN、NTK-aware 插值）

**Q3：如何将模型的上下文窗口从 4K 扩展到 128K？**

主流方案：
1. **位置编码插值**：将 RoPE 的基础频率从 10000 修改为更大的值（如 500000），拉伸位置编码的"分辨率"
2. **YaRN**：结合 NTK-aware 插值和注意力缩放，更平滑地扩展
3. **继续预训练**：在扩展后的位置编码上进行额外的预训练

## 标准答案

前馈神经网络（Feed-Forward Network, FFN）是 Transformer 中每一层的另一个核心组件，位于自注意力层之后。

**结构：**

```
FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
```

即两个线性变换中间夹一个 ReLU 激活函数（现代模型常用 SwiGLU 等变体）。

**作用：**

1. **特征变换与增强**：自注意力负责聚合全局上下文信息（"看什么"），FFN 负责对聚合后的信息进行非线性变换（"怎么用"），提升模型的表达能力。
2. **知识存储**：研究表明 FFN 是 Transformer 中存储事实知识的主要位置。每个 FFN 的第一层可以看作"键-值记忆"，W₁ 的每一行是一个"键"（匹配模式），W₂ 的每一列是一个"值"（对应输出）。
3. **维度变换**：FFN 将 d_model 维投影到 d_ff 维（通常是 4×d_model），再投影回来，相当于一个"信息瓶颈+扩展"的过程，提取更抽象的特征。
4. **非线性引入**：没有 FFN，Transformer 的多层堆叠将退化为线性变换（因为注意力的加权求和是线性的），FFN 引入的非线性是模型强大表达能力的关键。

## 深度解析

**FFN 作为"键值记忆"**

Geva et al. (2021) 的研究揭示了 FFN 的工作机理：FFN 的第一层 W₁ 的每一行 w₁ᵢ 可以看作一个"模式匹配器"（key），当输入 x 与 w₁ᵢ 的内积较大时，对应的 ReLU 激活为正。第二层 W₂ 的第 i 列 w₂ᵢ 可以看作该模式对应的"输出"（value）。因此：

```
FFN(x) = Σᵢ ReLU(x · w₁ᵢ) · w₂ᵢ
```

这类似于一个软检索机制——模型根据输入匹配不同的"知识条目"，加权输出。

**SwiGLU 变体**

原始 Transformer 使用 ReLU 激活，但现代大模型（Llama、PaLM、DeepSeek 等）普遍使用 SwiGLU：

```
SwiGLU(x) = (Swish(xW₁) ⊙ xW_gate)W₂
```

SwiGLU 通过门控机制（⊙ 为逐元素乘法）提供了更平滑的梯度，在相同参数量下性能优于 ReLU FFN。

## 🎯 面试追问

**Q1：为什么 FFN 的隐藏层维度通常是 d_model 的 4 倍？**

经验选择，最初 Transformer (d_model=512, d_ff=2048) 使用了 4 倍。这个比例在后续模型中被验证为有效的平衡点：
- 太小（2 倍）：表达能力不足
- 太大（8 倍）：参数量增加但收益递减
- 现代模型使用 SwiGLU 时，因为门控机制多了一组参数，通常将 d_ff 设为 8/3 × d_model（约 2.67 倍）来保持总参数量

**Q2：FFN 是存储知识的主要位置，这对模型编辑（Model Editing）有什么意义？**

这意味着可以通过修改 FFN 的权重来"编辑"模型的知识，而无需重新训练。具体技术包括：
- ROME（Rank-One Model Editing）：直接修改 FFN 的权重矩阵
- MEMIT：批量修改多个 FFN 层的权重
- 知识神经元定位：先找到存储特定知识的神经元，再定向修改

**Q3：MoE（Mixture of Experts）是如何利用 FFN 的？**

MoE 将 FFN 层替换为多个"专家"FFN，通过路由器（Router）选择激活哪些专家。这样：
- 总参数量很大（知识容量大）
- 每次推理只激活部分专家（计算量小）
- 是"以空间换时间"的典型应用

## 🚀 AI应用扩展

- **知识编辑**：在企业 AI 系统中，当产品信息更新时，可以直接编辑模型 FFN 中的知识，而不需要重新训练整个模型。
- **模型定制**：通过 LoRA 等技术微调 FFN 层的权重，可以在少量数据上快速适配特定业务场景。
- **MoE 部署**：在大规模推理系统中，MoE 架构通过只激活部分 FFN 专家，实现了"大参数量、低计算量"的推理。
- **模型压缩**：FFN 占据了 Transformer 参数量的约 2/3，因此 FFN 的压缩（剪枝、量化）是模型压缩的重点。

## ⚠️ 容易踩坑

1. **忽略 FFN 的重要性**：很多人只关注注意力机制，但 FFN 承担了知识存储和特征变换的关键角色，占总参数量的 2/3。
2. **混淆 FFN 在 Encoder 和 Decoder 中的差异**：Encoder 的 FFN 是双向的（可以看到所有位置），Decoder 的 FFN 是单向的（因果掩码）。
3. **误以为 FFN 是"简单的全连接层"**：FFN 在 Transformer 中承担着"键值记忆"的功能，其内部结构和训练后的权重包含了大量语义知识。

## 标准答案

残差连接（Residual Connection / Skip Connection）是 Transformer 中每个子层（Self-Attention 和 FFN）的关键设计，公式为：

```
output = LayerNorm(x + Sublayer(x))
```

即子层的输出不是直接作为下一层的输入，而是与输入 x 相加后再经过 Layer Normalization。

**解决的问题：**

1. **梯度消失/爆炸**：在深层网络中，梯度需要经过多层反向传播。残差连接提供了一条"梯度高速公路"——梯度可以跳过子层直接传回，缓解梯度消失问题。
2. **训练稳定性**：没有残差连接时，深层 Transformer（如 100+ 层）几乎无法训练。残差连接使得网络可以稳定训练到数百层。
3. **恒等映射学习**：残差连接使得网络只需学习输入的"增量"（残差），而非从头学习完整映射。如果某一层不需要变换，它只需要学习输出接近零的残差，保持恒等映射。
4. **特征重用**：浅层特征可以通过残差连接直接传到深层，避免信息丢失。

## 深度解析

**数学分析**

假设没有残差连接，第 l 层的输出为：
```
x_l = f_l(x_{l-1})
```

反向传播中梯度需要连乘：
```
∂Loss/∂x_0 = ∂Loss/∂x_L × ∏_{l=1}^{L} ∂f_l/∂x_{l-1}
```

当 L 很大时，如果 ∂f_l/∂x_{l-1} > 1（梯度爆炸）或 < 1（梯度消失），梯度将指数级增大或缩小。

加入残差连接后：
```
x_l = x_{l-1} + f_l(x_{l-1})
```

梯度变为：
```
∂Loss/∂x_0 = ∂Loss/∂x_L × ∏_{l=1}^{L} (1 + ∂f_l/∂x_{l-1})
```

由于始终有一个常数 1 存在，梯度不会完全消失。

**Pre-Norm vs Post-Norm**

- **Post-Norm**（原始 Transformer）：`LayerNorm(x + Sublayer(x))`——LayerNorm 在残差相加之后。训练不稳定，需要 warmup。
- **Pre-Norm**（GPT-2 及之后的主流方案）：`x + Sublayer(LayerNorm(x))`——LayerNorm 在子层之前。训练更稳定，支持更深的网络。

## 🎯 面试追问

Q1：残差连接和 Dropout 有什么关系？它们的作用是否重叠？

不重叠。残差连接解决的是梯度传播和恒等映射学习的问题；Dropout 解决的是过拟合问题。两者是互补的正则化/优化手段。

**Q2：为什么残差连接要加在子层而不是其他位置？**

残差连接加在子层是为了保证信息的"直通"路径——即使子层学习到的变换不好，信息仍然可以通过残差路径传递。如果残差连接加在其他位置，就无法保证这种信息直通。

**Q3：DeepNet 等超深 Transformer 是如何解决残差连接的数值问题的？**

DeepNet 提出了 DeepNorm，在残差连接中引入缩放因子：
```
output = LayerNorm(α · x + Sublayer(x))
```
其中 α > 1，用于放大残差路径的信号。同时初始化时将子层的权重缩小。这使得模型可以稳定训练到 1000+ 层。

## 🚀 AI应用扩展

- **模型训练**：在企业级模型训练中，残差连接是保证训练稳定性的关键。特别是在大规模分布式训练中，梯度的稳定传播对收敛至关重要。
- **LoRA 微调**：LoRA 的核心思想与残差连接类似——在预训练权重旁边添加一个低秩增量，而不是修改原始权重。这可以看作"残差连接"思想在微调中的应用。
- **模型编辑**：通过在模型中插入额外的残差模块（如 Adapter），可以快速适配新任务，而无需修改原始模型权重。
- **特征融合**：在多模态模型中，残差连接用于融合不同模态的特征，确保原始信息不会在融合过程中丢失。

## ⚠️ 容易踩坑

1. **忽略残差连接对训练稳定性的关键作用**：没有残差连接，深层 Transformer 几乎无法训练。
2. **混淆 Pre-Norm 和 Post-Norm 的区别**：两者的 LayerNorm 位置不同，训练行为差异很大。
3. **误以为残差连接只在 Transformer 中使用**：ResNet（2015）在 CNN 中首先大规模使用残差连接，Transformer 借鉴了这一设计。

## 标准答案

GPT（Generative Pre-trained Transformer）选择纯解码器（Decoder-Only）架构，核心原因如下：

1. **天然适配自回归生成**：纯解码器使用因果掩码（Causal Mask），确保每个位置只能看到之前的位置，完美匹配自回归生成任务的"逐 token 预测"范式。

2. **训练效率更高**：每次前向传播中，序列中的每个位置都可以作为预测目标（Next Token Prediction），n 个 token 产生 n-1 个训练信号。而 Encoder-Decoder 架构（如 T5）的训练信号密度相对较低。

3. **架构简洁统一**：同一个模型既能理解（编码）又能生成（解码），无需维护两套参数。这简化了模型设计、训练流程和推理部署。

4. **涌现能力（Emergent Abilities）**：实验证明 Decoder-Only 架构在模型规模增大时更容易出现 In-Context Learning、Chain-of-Thought 等涌现能力。

5. **推理优化友好**：自回归生成可以利用 KV Cache 技术，每次只需计算新 token 的注意力，历史 token 的 KV 值可以缓存复用，推理效率高。

## 深度解析

**三种 Transformer 架构对比**

| 架构 | 代表模型 | 注意力模式 | 适用任务 |
|------|---------|-----------|---------|
| Encoder-Only | BERT | 双向（全注意力） | 理解/分类/抽取 |
| Encoder-Decoder | T5, BART | 编码双向 + 解码因果 | Seq2Seq（翻译、摘要） |
| Decoder-Only | GPT, Llama | 因果（单向） | 生成/对话/通用 |

**为什么 Decoder-Only 在大规模模型中表现最好？**

- **Scaling Law 友好**：研究表明，在相同参数量和数据量下，Decoder-Only 架构的 loss 下降更快
- **双向注意力的"信息泄露"问题**：BERT 式的双向注意力在预训练时可以看到被 mask 的词的上下文，但这与下游生成任务不匹配（推理时看不到未来信息）。这种训练-推理不一致会限制模型能力
- **通用性**：通过适当的 prompt 设计，Decoder-Only 模型可以模拟理解和生成两种能力

## 🏠 生活类比

三种架构就像三种不同的**阅读方式**：

- **Encoder-Only（BERT）**：完形填空式阅读——把书中的某些词涂黑，根据上下文猜。你可以看到前后文。
- **Encoder-Decoder（T5）**：翻译式阅读——先完整理解英文原文（编码器），再逐词翻译成中文（解码器）。
- **Decoder-Only（GPT）**：接龙式阅读——从第一个词开始，一个词一个词地往下猜。你只能看到已经猜过的词，不能偷看后面的。

GPT 选择"接龙"模式，因为它最接近人类创作文字的方式——一个字一个字地写，每个字都基于之前写过的内容。

## 🎯 面试追问

**Q1：BERT 和 GPT 哪个更适合做文本分类？**

理论上 BERT 更适合，因为双向注意力可以同时看到上下文。但实践证明，GPT 在 prompt 设计得当的情况下，分类能力也不差，甚至在 few-shot 场景下更灵活。在大模型时代（2024-2026），趋势是用 Decoder-Only 模型（如 GPT-4、Llama）统一处理所有任务。

**Q2：因果掩码如何实现？**

在自注意力的 Score 矩阵上，将未来位置的分数设为 -∞：
```python
mask = torch.triu(torch.ones(n, n), diagonal=1).bool()
scores = scores.masked_fill(mask, float('-inf'))
```
softmax(-∞) = 0，确保未来位置的注意力权重为 0。

**Q3：既然 Encoder-Decoder 架构参数量更大，为什么反而不如 Decoder-Only？**

参数量大不等于能力强。Encoder-Decoder 的参数分布在编码器和解码器两部分，训练信号需要通过 Cross-Attention 桥接，优化路径更长。Decoder-Only 的参数全部用于同一个自回归目标，优化更高效。此外，Scaling Law 表明在固定计算预算下，Decoder-Only 架构的 loss 下降更快。

- **多轮推理**：在 Chain-of-Thought 等推理场景中，模型逐步生成推理过程，每一步都基于之前的推理结果。

## ⚠️ 容易踩坑

1. **误以为 Decoder-Only 不能做理解任务**：通过适当的 prompt 设计（如"请判断以下文本的情感倾向"），Decoder-Only 模型可以完成各种理解任务。
2. **混淆因果掩码和填充掩码**：因果掩码防止看到未来 token，填充掩码（padding mask）处理不同长度的输入序列，两者需要同时使用。
3. **忽略 Encoder-Decoder 在特定任务上的优势**：在翻译、摘要等强 Seq2Seq 任务上，Encoder-Decoder 架构（如 T5、Flan-T5）可能仍然优于同参数量的 Decoder-Only 模型。

## 标准答案

自回归模型（Autoregressive Model）是一类概率生成模型，它将联合概率分布分解为条件概率的连乘：

```
P(x₁, x₂, ..., xₙ) = P(x₁) × P(x₂|x₁) × P(x₃|x₁,x₂) × ... × P(xₙ|x₁,...,xₙ₋₁)
                = ∏_{t=1}^{n} P(xₜ | x₁, ..., xₜ₋₁)
```

在语言模型中，这意味着每个 token 的生成都基于之前所有 token 的条件概率。

**核心特征：**

1. **逐 token 生成**：模型按顺序生成文本，每次预测一个 token
2. **因果依赖**：每个 token 只依赖于它前面的 token（因果性）
3. **自回归性质**：输出作为下一步的输入（"自回归"——自己回归自己）
4. **Teacher Forcing 训练**：训练时使用真实的前文（而非模型生成的前文）作为条件

**代表模型**：GPT 系列、Llama 系列、Qwen 系列、DeepSeek 系列

## 深度解析

**自回归 vs 非自回归**

| 特性 | 自回归（AR） | 非自回归（NAR） |
|------|-------------|----------------|
| 生成方式 | 逐步生成 | 并行生成 |
| 生成质量 | 高（考虑完整上下文） | 低（独立生成各 token） |
| 生成速度 | 慢（O(n) 步） | 快（O(1) 步） |
| 长度确定 | 动态（直到 EOS） | 需要预先确定 |
| 代表模型 | GPT、Llama | NAT、Mask-Predict |

**自回归模型的生成策略**

解码策略直接影响生成质量：
- **贪心解码**：每次选概率最高的 token。快速但容易重复。
- **束搜索（Beam Search）**：同时维护 k 个候选序列。质量更好但更慢。
- **采样解码**：从概率分布中随机采样。多样性好但可能不连贯。
- **Top-k / Top-p 采样**：限制采样范围，平衡多样性和质量。
- **Temperature**：控制概率分布的"尖锐度"。低温趋近贪心，高温增加随机性。

## 🎯 面试追问

**Q1：自回归模型的训练和推理有什么区别？**

- **训练**（Teacher Forcing）：使用真实的前文作为输入，模型学习预测下一个 token。所有位置可以并行计算（因为前文是已知的）。
- **推理**（Auto-regressive Decoding）：使用模型生成的前文作为输入，逐 token 生成。每次生成一个 token 后才能生成下一个。

这种"训练-推理不一致"被称为 Exposure Bias。

**Q2：什么是 Exposure Bias？如何缓解？**

训练时用真实前文，推理时用模型自己生成的前文。如果模型在某一步生成了错误的 token，错误会累积。

缓解方法：
- Scheduled Sampling：训练时逐渐从使用真实前文过渡到使用模型生成的前文
- 强化学习（RLHF）：通过人类反馈直接优化生成质量
- Beam Search / Nucleus Sampling：通过更好的解码策略减少错误累积

**Q3：自回归模型为什么不能双向关注？**

自回归模型的任务是 P(xₜ | x₁, ..., xₜ₋₁)，即根据过去预测未来。如果允许双向关注，模型在训练时就能"看到答案"（未来的 token），这与推理时无法看到未来的设定矛盾，会导致训练-推理不一致。

## 🚀 AI应用扩展

- **流式输出**：自回归的逐步生成特性天然支持流式输出（Streaming），用户可以在模型生成过程中实时看到结果，提升体验。
- **代码补全**：IDE 中的代码补全是自回归模型的典型应用——模型根据已输入的代码逐 token 预测后续代码。
- **自动摘要**：给定长文档，自回归模型逐步生成摘要文本。
- **对话系统**：每轮对话的回复都是自回归生成的，模型根据完整的对话历史生成下一步回复。

## ⚠️ 容易踩坑

1. **混淆自回归和序列到序列**：自回归模型（如 GPT）是单向的——只有解码器。Seq2Seq 模型（如 T5）有编码器和解码器两部分。
2. **忽略 Teacher Forcing 的影响**：训练时使用真实前文会导致 Exposure Bias，这是自回归模型的一个已知问题。
3. **误以为自回归模型只能做生成任务**：通过 prompt 设计，自回归模型可以完成分类、抽取、翻译等各种任务。

## 标准答案

**NTP（Next Token Prediction）** 和 **MLM（Masked Language Modeling）** 是两种核心的预训练目标：

| 特性 | NTP（下一句预测） | MLM（掩码语言模型） |
|------|------------------|-------------------|
| 代表模型 | GPT 系列 | BERT 系列 |
| 预测方式 | 预测下一个 token | 预测被 mask 的 token |
| 上下文方向 | 单向（只看左侧） | 双向（看左右两侧） |
| 训练信号密度 | n-1 个预测目标 | ~15% 的 token（约 0.15n） |
| 掩码策略 | 因果掩码（上三角） | 随机 mask 15% token |
| 生成能力 | ✅ 天然支持 | ❌ 不支持自回归生成 |
| 理解能力 | ✅ 通过 prompt | ✅ 天然擅长 |

**MLM 的具体做法**（BERT）：
1. 随机选择 15% 的 token
2. 其中 80% 替换为 [MASK]，10% 替换为随机词，10% 保持不变
3. 模型预测原始 token

**NTP 的具体做法**（GPT）：
1. 使用因果掩码，每个位置只能看到之前的 token
2. 模型预测下一个 token
3. 所有位置都参与损失计算

## 深度解析

**为什么 MLM 的训练信号密度低？**

MLM 只对 15% 的被 mask token 计算损失，而 NTP 对序列中每个位置都计算损失。这意味着：
- 在相同的训练数据量下，NTP 每次前向传播提供的训练信号是 MLM 的约 6.7 倍
- MLM 需要更多训练步数才能达到相同的训练信号量

**为什么 MLM 使用双向上下文？**

被 mask 的 token 可以同时关注它左边和右边的上下文。例如：

```
输入: The [MASK] sat on the mat
```

模型可以同时利用 "The" 和 "sat on the mat" 来预测 "cat"。

而 NTP 只能关注左侧：
```
输入: The cat sat on the
模型预测: mat  ← 只能利用 "The cat sat on the"
```

**为什么 NTP 在大规模模型中更成功？**

1. **训练效率**：NTP 的训练信号密度高，在大规模预训练中更高效
2. **自回归一致性**：NTP 的训练和推理模式一致（都是从左到右），不存在 MLM 的预训练-微调不一致
3. **涌现能力**：大规模 NTP 模型更容易出现 In-Context Learning、Chain-of-Thought 等涌现能力
4. **通用性**：通过 prompt，NTP 模型可以完成几乎所有 MLM 能做的任务

## 🏠 生活类比

- **NTP** 就像**接龙游戏**：每个人只能说一个字，必须基于前面已经说的字来接。你不能偷看后面的字。
- **MLM** 就像**完形填空考试**：试卷上有些词被涂黑了，你需要根据上下文（前后文）来猜被涂黑的词是什么。

接龙游戏更考验"创作能力"（生成），完形填空更考验"理解能力"（理解）。但随着能力的提升，擅长接龙的人也可以做好完形填空（大模型时代的趋势）。

## 🎯 面试追问

**Q1：为什么 BERT 不能做自回归生成？**

BERT 的 MLM 训练目标要求模型同时看到左右上下文。在推理时，如果要生成文本，右边的 token 还不存在，模型无法利用右侧上下文，这与训练时的模式不一致。虽然可以通过 iterative decoding（如 Mask-Predict）实现非自回归生成，但效果远不如 GPT。

**Q2：有没有结合 NTP 和 MLM 优点的方案？**

有：
- **Prefix LM**（如 UniLM）：前缀部分使用双向注意力，生成部分使用因果注意力
- **XLNet**：使用排列语言模型（Permutation LM），在保持自回归性质的同时实现双向上下文
- **SpanBERT**：mask 连续的 span 而非单个 token
- **T5**：使用 Span Corruption 预训练目标

**Q3：在大模型时代（2024-2026），BERT 还有价值吗？**

有价值，但在缩小：
- **Embedding 任务**：BERT 仍是最好的文本编码器之一（如 BGE、E5 系列）
- **轻量级任务**：在分类、NER 等轻量级任务上，BERT 系列模型（如 DeBERTa）仍然高效
- **RAG 系统**：检索阶段的向量编码通常使用 BERT 类模型
- **端侧部署**：在资源受限的场景下，小规模 BERT 模型仍然实用

## 🚀 AI应用扩展

- **RAG 系统中的混合使用**：检索阶段使用 MLM 模型（BERT 编码器）生成文档向量，生成阶段使用 NTP 模型（GPT 解码器）生成回答。
- **意图识别**：在智能客服中，快速的 BERT-based 分类器用于意图识别，GPT-based 模型用于回复生成。
- **知识蒸馏**：用大 NTP 模型（GPT-4）生成训练数据，蒸馏到小 MLM 模型（BERT）中，兼顾效率和质量。
- **多任务预训练**：结合 NTP 和 MLM 的混合预训练目标（如 T5、UL2），在多种下游任务上都有好的表现。

## ⚠️ 容易踩坑

1. **混淆 NTP 和 NSP**：NTP 是 Next Token Prediction（预测下一个 token），NSP 是 Next Sentence Prediction（预测下一句）。NSP 是 BERT 的辅助任务，已被证明不太有效。
2. **误以为 MLM 可以替代 NTP**：MLM 不能做自回归生成，两种预训练目标有本质区别。
3. **忽略 MLM 的预训练-微调不一致**：训练时有 [MASK] token，推理时没有，这种不一致会影响模型性能。

```
║  RoPE (原始)     │   4K     │   8K     │ +15.2 (严重退化)       ║
║  RoPE + YaRN     │   4K     │  128K    │ +0.8  (几乎无损)       ║
║  RoPE + NTK      │   4K     │   32K    │ +3.5  (轻微退化)       ║
║  ALiBi           │   1K     │   6K     │ +1.2  (较好外推)       ║
║  ALiBi           │   1K     │  16K     │ +8.7  (明显退化)       ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝
```

### RoPE 的复数空间直觉

RoPE 将 d_k 维向量视为 d_k/2 个二维平面上的点，对每个平面施加角度为 m·θ_i 的旋转：

```
q_m = [q₁cos(mθ₁) - q₂sin(mθ₁), q₁sin(mθ₁) + q₂cos(mθ₁), ...]
```

两个旋转向量的点积只依赖角度差 (m-n)·θ，因此天然编码了相对位置。

## 专题三：Flash Attention v2/v3

### 核心问题

标准自注意力需要存储完整的 n×n 注意力矩阵，空间复杂度 O(n²)。对于长序列（n=128K），这需要数十 GB 显存。

### Flash Attention 核心思想

```
╔══════════════════════════════════════════════════════════════════╗
║              Flash Attention 工作原理                             ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║  标准 Attention:                                                 ║
║  1. 计算 QK^T (n×n 矩阵，写入 HBM)                              ║
║  2. Softmax (读取 n×n，写入 HBM)                                 ║
║  3. 乘以 V (读取 n×n 和 V，写入结果)                             ║
║  → 大量 HBM 读写，带宽瓶颈                                       ║
║                                                                  ║
║  Flash Attention:                                                ║
║  1. 将 Q、K、V 分块 (Tiling)                                     ║
║  2. 每个块在 SRAM 中完成 QK^T → Softmax → ×V                     ║
║  3. 使用在线 Softmax 算法，逐块合并结果                           ║
║  4. 不需要存储完整的 n×n 矩阵                                    ║
║  → 空间复杂度 O(n)，HBM 访问量大幅减少                           ║
║                                                                  ║
║  SRAM (快但小) vs HBM (慢但大):                                  ║
║  ┌──────────────────────────┐                                    ║
║  │ SRAM: 20MB, 19TB/s       │ ← Flash Attention 在这里计算       ║
║  │ HBM:  40GB, 1.5TB/s      │ ← 标准 Attention 频繁读写          ║
║  └──────────────────────────┘                                    ║
║  Flash Attention 减少 HBM 访问 → 速度提升 2-4x                   ║
║                                                                  ║
╠══════════════════════════════════════════════════════════════════╣
║  版本演进:                                                       ║
║                                                                  ║
║  Flash Attention v1 (2022):                                      ║
║  - 分块计算 + 在线 Softmax + 重计算                              ║
║  - 空间 O(n)，速度提升 2-4x                                      ║
║                                                                  ║
║  Flash Attention v2 (2023):                                      ║
║  - 优化并行策略（序列维度并行）                                   ║
║  - 减少非矩阵乘法运算                                            ║
║  - 速度再提升 2x，接近理论峰值                                    ║
║                                                                  ║
║  Flash Attention v3 (2024):                                      ║
║  - 针对 H100 优化（利用 FP8、TMA）                               ║
║  - 异步执行 Warp 级流水线                                         ║
║  - 在 H100 上达到 740 TFLOPS（理论峰值的 75%）                   ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝
```

### 面试关键点

- Flash Attention **不改变计算结果**（数学上等价），只改变计算顺序
- 核心是 IO-aware 算法设计——优化 HBM 和 SRAM 之间的数据搬运
- 重计算（Recomputation）：反向传播时不存储注意力矩阵，而是重新计算

## 专题四：KV Cache 优化

### 问题背景

自回归生成时，每生成一个新 token 都需要计算它与所有历史 token 的注意力。如果每次都重新计算所有 token 的 K 和 V，会非常低效。

### KV Cache 原理

```
╔══════════════════════════════════════════════════════════════════╗
║              KV Cache 工作原理                                    ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║  无 KV Cache (每步重新计算):                                     ║
║  步骤 1: 计算 [x₁] 的 K₁, V₁                                    ║
║  步骤 2: 计算 [x₁,x₂] 的 K₁,K₂, V₁,V₂  ← K₁ 重复计算          ║
║  步骤 3: 计算 [x₁,x₂,x₃] 的 K₁,K₂,K₃, V₁,V₂,V₃ ← 重复        ║
║  → 计算量 O(n²)                                                  ║
║                                                                  ║
║  有 KV Cache (缓存历史):                                         ║
║  步骤 1: 计算 [x₁] 的 K₁, V₁ → 缓存                            ║
║  步骤 2: 从缓存读 K₁, 只计算 K₂, V₂ → 追加缓存                  ║
║  步骤 3: 从缓存读 K₁,K₂, 只计算 K₃, V₃ → 追加缓存              ║
║  → 计算量 O(n)，缓存复用                                        ║
║                                                                  ║
╠══════════════════════════════════════════════════════════════════╣
║  KV Cache 显存占用公式:                                          ║
║                                                                  ║
║  显存 = 2 × n_layers × n_heads × d_head × seq_len × batch_size  ║
║        × bytes_per_element                                       ║
║                                                                  ║
║  示例 (Llama 2 70B, seq_len=4096, batch=1, FP16):               ║
║  = 2 × 80 × 64 × 128 × 4096 × 1 × 2 bytes                      ║
║  = 5.37 GB                                                       ║
║  → 几乎占满一张 A100 80GB 的相当比例                              ║
║                                                                  ║
╠══════════════════════════════════════════════════════════════════╣
║  KV Cache 优化技术:                                              ║
║                                                                  ║
║  1. GQA/MQA: 减少 KV Head 数量 → KV Cache 按比例缩小            ║
║  2. KV Cache 量化: 将 FP16 → INT8/INT4 → 显存减半/四分之一      ║
║  3. PagedAttention (vLLM): 按页管理 KV Cache，减少碎片           ║
║  4. Token 淘汰: 淘汰不重要的历史 token（如 H2O、StreamingLLM）  ║
║  5. 共享前缀: 多个请求共享 system prompt 的 KV Cache             ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝
```

### MoE 的工程挑战

1. **All-to-All 通信**：不同专家可能在不同 GPU 上，需要跨卡通信

4. **训练不稳定**：路由策略可能导致训练振荡

## 专题六：Mamba / State Space Models 等新架构

### Mamba 核心思想

Mamba 是一种基于选择性状态空间模型（Selective SSM）的序列建模架构，被认为是 Transformer 的潜在竞争者。

```
╔══════════════════════════════════════════════════════════════════╗
║              Transformer vs Mamba 对比                           ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║  Transformer:                                                    ║
║  - 注意力机制: O(n²) 复杂度                                      ║
║  - 全局视野: 每个 token 可以关注所有其他 token                    ║
║  - KV Cache: 推理时需要线性增长的缓存                             ║
║  - 训练: 高度并行，GPU 友好                                      ║
║                                                                  ║
║  Mamba:                                                          ║
║  - 选择性 SSM: O(n) 复杂度                                       ║
║  - 递归计算: 通过隐状态传递信息                                   ║
║  - 无 KV Cache: 推理时显存恒定                                    ║
║  - 训练: 利用并行扫描算法，GPU 友好                               ║
║                                                                  ║
╠══════════════════════════════════════════════════════════════════╣
║  Mamba 的选择性机制:                                             ║
║                                                                  ║
║  传统 SSM: A, B, C 是固定的 → 线性时不变系统                      ║
║  Mamba SSM: A, B, C 依赖输入 x → 输入依赖的选择性机制             ║
║                                                                  ║
║  xₜ → [B(xₜ), C(xₜ), Δ(xₜ)] → 选择性地记住/遗忘信息           ║
║                                                                  ║
║  直觉: 模型可以动态决定"保留什么信息"和"丢弃什么信息"            ║
║  类似 LSTM 的门控，但在连续时间域上操作                           ║
║                                                                  ║
╠══════════════════════════════════════════════════════════════════╣
║  性能对比 (2024 基准):                                           ║
║                                                                  ║
║  任务类型          │ Transformer │ Mamba   │ 胜出               ║
║  ──────────────────┼─────────────┼─────────┼───────             ║
║  语言建模 (PPL)    │    优秀     │  接近   │ Transformer 略胜   ║
║  长序列 (>64K)     │    受限     │  优秀   │ Mamba              ║
║  推理吞吐          │    受 KV 限制│  优秀   │ Mamba              ║
║  In-Context Learn  │    优秀     │  较弱   │ Transformer        ║
║  复制/检索任务     │    优秀     │  较弱   │ Transformer        ║
║                                                                  ║
║  趋势: 混合架构 (Jamba = Mamba + Transformer) 成为主流           ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝
```

### Jamba：混合架构的探索

AI21 的 Jamba 模型将 Mamba 层和 Transformer 层交错堆叠：
- Mamba 层负责高效的长距离信息传递
- Transformer 层负责精确的全局信息检索
- 兼顾效率和能力

### Mamba 2 与 SSD (State Space Duality)

Mamba 2 发现 SSM 和注意力机制之间存在数学对偶性：
- SSM 可以用矩阵乘法实现（类似注意力）
- 注意力可以看作一种特殊的 SSM
- 这为统一两种范式提供了理论基础

# 面试前5分钟冲刺版

**必背公式：**
- `Attention(Q,K,V) = softmax(QK^T / √d_k) · V`
- `FFN(x) = max(0, xW₁ + b₁)W₂ + b₂`（SwiGLU: `Swish(xW₁) ⊙ xW_gate)W₂`）
- `PE(pos,2i) = sin(pos/10000^(2i/d))`, `PE(pos,2i+1) = cos(pos/10000^(2i/d))`

**必记概念：**
1. **Self-Attention** = Q找K匹配 → Softmax归一化 → 加权聚合V
2. **Multi-Head** = h个子空间并行注意力 → Concat → W_O投影
3. **位置编码** = 注入位置信息（自注意力本身无位置感知）
4. **FFN** = 特征变换 + 知识存储（占参数量2/3）
5. **残差连接** = 梯度高速公路（x + Sublayer(x)）
6. **Decoder-Only** = 因果掩码 + NTP + 统一理解与生成

**必答追问：**
- "MHA vs MQA vs GQA？" → KV共享程度不同，GQA是折中
- "Flash Attention？" → 分块计算 + 在线Softmax + 不存n²矩阵
- "RoPE？" → 旋转编码相对位置，配合YaRN外推
- "MoE？" → 多专家路由，大参数小计算
- "Mamba？" → 选择性SSM，O(n)复杂度，混合架构趋势

**一句话总结：**
> Transformer 用自注意力解决了 RNN 的长距离依赖和并行化问题，通过 Q/K/V 分离、多头并行、位置编码、FFN 知识存储和残差连接构建了强大的序列建模架构，Decoder-Only 变体（GPT 系列）在 Scaling Law 的推动下成为大模型时代的主流选择。

### 面试前5分钟快速复习

**必背概念TOP10：**
1. 自注意力机制（Self-Attention）：每个Token与序列中所有Token计算相似度，动态加权聚合信息
2. 多头注意力（Multi-Head Attention）：将Q/K/V拆分到多个头并行计算，捕获不同子空间的关系
3. 位置编码（Positional Encoding）：Transformer无天然顺序感知，需注入位置信息（正弦/旋转RoPE）
4. LayerNorm：层归一化，稳定训练过程，Pre-Norm（现代）vs Post-Norm（原始）
5. 残差连接（Residual Connection）：解决深层网络梯度消失，y = F(x) + x
6. FFN（Feed-Forward Network）：两层全连接+激活函数，存储知识的关键组件
7. Encoder-Decoder结构：原始Transformer的双模块设计，Encoder理解+Decoder生成
8. Decoder-Only变体：GPT系列采用，因果掩码确保自回归生成，2026年主流架构
9. Scaling Law：模型性能随参数量、数据量、算力的幂律增长关系
10. MoE（Mixture of Experts）：稀疏激活，大参数小计算，提升模型容量而不等比增加推理成本

**必背公式/参数：**
- Attention公式：Attention(Q,K,V) = softmax(QK^T / √d_k) × V
- √d_k缩放因子：防止点积过大导致softmax梯度消失
- 多头输出：MultiHead = Concat(head_1, ..., head_h) × W^O
- FFN公式：FFN(x) = max(0, xW_1 + b_1)W_2 + b_2（ReLU激活）
- RoPE位置编码：旋转矩阵编码相对位置，支持长度外推

**必会架构图：**
```
输入Token → Embedding + Positional Encoding
    ↓
┌─────────────────────────────────┐
│  Multi-Head Self-Attention       │
│  Q=XW_Q, K=XW_K, V=XW_V        │
│  Attn = softmax(QK^T/√d_k)V    │
│  Output = Concat(heads) × W_O   │
├─────────────────────────────────┤
│  Add & LayerNorm (残差连接)      │
├─────────────────────────────────┤
│  Feed-Forward Network            │
│  FFN(x) = ReLU(xW1+b1)W2+b2    │
├─────────────────────────────────┤
│  Add & LayerNorm (残差连接)      │
└─────────────────────────────────┘
    ↓
输出（送入下一层或最终预测）
```

**核心口诀速查：**
- 注意力三步走：算相似度(QK^T) → 归一化(softmax) → 加权聚合(×V)
- 三类位置编码：正弦(原始) → RoPE(旋转,主流) → ALiBi(线性偏置)
- 两大架构流派：Encoder-Decoder(T5/BART) vs Decoder-Only(GPT/LLaMA)
- 训练三大技巧：残差连接 + LayerNorm + 学习率Warmup

## ⚡ 面试前5分钟冲刺

**Q: 自注意力机制的核心思想？**
30秒答：让序列中每个Token都能直接关注其他所有Token，通过Q·K^T计算相似度，softmax归一化后加权聚合V，解决RNN的长距离依赖问题。

**Q: 为什么要除以√d_k？**
30秒答：防止点积值过大导致softmax输出接近one-hot，梯度趋近于零。缩放后方差保持为1，梯度更稳定。

**Q: 多头注意力的作用？**
30秒答：将注意力拆分到多个子空间并行计算，不同头可学习不同类型的关系（语法、语义、位置等），最后拼接融合。

**Q: Pre-Norm和Post-Norm的区别？**
30秒答：Pre-Norm在子层之前做LayerNorm，梯度更稳定，训练更容易收敛，是现代LLM的主流选择；Post-Norm在子层之后做，原始Transformer设计。

**Q: RoPE位置编码的优势？**
30秒答：通过旋转矩阵编码相对位置关系，天然支持相对位置建模，且具有良好的长度外推能力（配合NTK-aware插值）。

**Q: 为什么Decoder-Only成为主流？**
30秒答：1) 统一的因果掩码训练范式，简单高效；2) Scaling Law验证其随规模增长性能提升更稳定；3) 适配自回归生成任务。

**Q: MoE的原理和优势？**
30秒答：多个专家网络+路由器，每个Token只激活Top-K个专家。优势：参数量大但计算量小，如Switch Transformer、Mixtral。

**Q: FFN在Transformer中的作用？**
30秒答：FFN是知识存储的主要组件。研究发现FFN的第一层存储"键"，第二层存储"值"，类似于一个可微的记忆网络。

**Q: Transformer的计算复杂度？**
30秒答：自注意力O(n²d)，FFN O(nd²)，n为序列长度，d为维度。长序列瓶颈在注意力，可用Flash Attention优化。

**Q: 什么是KV Cache？为什么重要？**
30秒答：自回归生成时缓存已计算的K和V矩阵，避免每步重新计算整个序列的注意力，将推理从O(n²)降到O(n)。

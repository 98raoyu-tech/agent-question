### 一、项目背景与需求分析

企业级RAG（Retrieval-Augmented Generation）知识库项目的核心目标是将企业内部海量非结构化文档（PDF、Word、Excel、Markdown等）转化为可被大语言模型高效检索和引用的知识资产。

**典型业务场景**：
- **企业内部知识问答**：员工可以自然语言查询公司制度、技术文档、产品手册等
- **客户服务支撑**：客服人员快速检索产品知识，提供精准回答
- **合规与法务支持**：快速检索合同条款、法规条文，辅助风险评估
- **研发知识沉淀**：技术文档、API文档、故障排查手册的智能检索

**核心需求拆解**：

```
功能性需求：
├── 多格式文档解析（PDF/Word/Excel/PPT/HTML/Markdown）
├── 智能分块策略（语义分块/递归分块/文档结构分块）
├── 高精度检索（向量检索 + 关键词检索 + 混合检索）
├── 多轮对话支持（上下文感知的查询改写）
├── 答案溯源（标注答案来源文档及段落）
└── 权限控制（不同角色访问不同文档集合）

非功能性需求：
├── 检索延迟 < 2s（P99）
├── 答准确率 > 85%
├── 支持 100万+ 文档段落索引
├── 支持并发 50+ 用户同时查询
└── 文档增量更新，无需全量重建索引
```

### 二、技术架构设计

整体架构采用分层设计，从前端交互层到后端服务层，再到数据存储层和模型层，每一层职责清晰、可独立扩展。

**前端层**：
- React/Vue 3 + TypeScript 构建的对话界面
- 支持富文本渲染、Markdown展示、代码高亮
- 答案来源引用的可视化展示（高亮定位到原文段落）
- 文档管理后台（上传、预览、权限配置）

**后端服务层**：
- Python FastAPI / Node.js 提供 RESTful API
- LangChain / LlamaIndex 作为 RAG 编排框架
- Celery 异步任务队列处理文档解析和向量化
- Redis 缓存热点查询结果

**数据存储层**：
- 向量数据库：Milvus / Qdrant / Weaviate（存储文档嵌入向量）
- 关系型数据库：PostgreSQL（存储文档元数据、用户权限、对话历史）
- 对象存储：MinIO / S3（存储原始文档文件）
- 搜索引擎：Elasticsearch（全文检索、关键词匹配）

**模型层**：
- Embedding 模型：bge-large-zh-v1.5 / text-embedding-3-small

**第一阶段：文档解析**

```python
"""文档解析器 - 支持多格式统一解析"""
import os
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from pathlib import Path
from typing import Optional


@dataclass
class ParsedDocument:
    """解析后的文档结构"""
    content: str
    metadata: dict = field(default_factory=dict)
    page_count: int = 0
    file_type: str = ""
    file_name: str = ""
    file_size: int = 0


class BaseDocumentParser(ABC):
    """文档解析器基类"""

    @abstractmethod
    def parse(self, file_path: str) -> ParsedDocument:
        """解析文档，返回统一格式的文档内容"""
        ...

    @abstractmethod
    def supported_extensions(self) -> list[str]:
        """支持的文件扩展名"""
        ...


class PDFParser(BaseDocumentParser):
    """PDF文档解析器"""

    def __init__(self):
        # 使用 PyMuPDF (fitz) 进行PDF解析
        import fitz
        self._fitz = fitz

    def supported_extensions(self) -> list[str]:
        return [".pdf"]

    def parse(self, file_path: str) -> ParsedDocument:
        doc = self._fitz.open(file_path)
        pages_content = []

        for page_num in range(len(doc)):
            page = doc[page_num]
            # 提取文本内容
            text = page.get_text("text")
            # 提取表格（使用 camelot 或 tabula）
            tables = self._extract_tables(page)
            # 组合页面内容
            page_content = text
            if tables:
                page_content += "\n\n" + "\n".join(tables)
            pages_content.append({
                "page_num": page_num + 1,
                "content": page_content.strip()
            })

        doc.close()

        return ParsedDocument(
            content="\n\n".join(p["content"] for p in pages_content),
            metadata={"pages": pages_content},
            page_count=len(pages_content),
            file_type="pdf",
            file_name=Path(file_path).name,
            file_size=os.path.getsize(file_path)
        )

    def _extract_tables(self, page) -> list[str]:
        """从PDF页面提取表格并转换为Markdown格式"""
        # 简化实现 - 实际项目中使用 camelot-py 或 pdfplumber
        tables = page.find_tables()
        result = []
        for table in tables:
            md_table = self._table_to_markdown(table)
            result.append(md_table)
        return result

    def _table_to_markdown(self, table) -> str:
        """将表格数据转换为Markdown格式"""
        data = table.extract()
        if not data:
            return ""
        header = "| " + " | ".join(str(cell) for cell in data[0]) + " |"
        separator = "| " + " | ".join("---" for _ in data[0]) + " |"
        rows = [
            "| " + " | ".join(str(cell) for cell in row) + " |"
            for row in data[1:]
        ]
        return "\n".join([header, separator] + rows)


class DocumentParserFactory:
    """文档解析器工厂 - 根据文件类型选择合适的解析器"""

    def __init__(self):
        self._parsers: dict[str, BaseDocumentParser] = {}

    def register(self, parser: BaseDocumentParser) -> None:
        """注册解析器"""
        for ext in parser.supported_extensions():
            self._parsers[ext.lower()] = parser

    def parse(self, file_path: str) -> ParsedDocument:
        """解析文档"""
        ext = Path(file_path).suffix.lower()
        parser = self._parsers.get(ext)
        if not parser:
            raise ValueError(f"不支持的文件格式: {ext}")
        return parser.parse(file_path)


# 初始化工厂并注册所有解析器
def create_parser_factory() -> DocumentParserFactory:
    factory = DocumentParserFactory()
    factory.register(PDFParser())
    # factory.register(WordParser())
    # factory.register(ExcelParser())
    # factory.register(MarkdownParser())
    return factory
```

**第二阶段：智能分块**

分块策略是RAG系统的核心环节，分块粒度直接影响检索精度和回答质量。

```python
"""智能分块策略 - 多种分块方式支持"""
from dataclasses import dataclass
from typing import Optional
import re


@dataclass
class DocumentChunk:
    """文档分块"""
    content: str
    metadata: dict
    chunk_index: int
    token_count: int
    start_char: int
    end_char: int


class RecursiveCharacterSplitter:
    """递归字符分块器 - 最常用的通用分块策略"""

    def __init__(
        self,
        chunk_size: int = 512,
        chunk_overlap: int = 64,
        separators: Optional[list[str]] = None,
        length_function=None
    ):
        self._chunk_size = chunk_size
        self._chunk_overlap = chunk_overlap
        self._separators = separators or ["\n\n", "\n", "。", "！", "？", ".", "!", "?", "；", ";", " "]
        self._length_function = length_function or len

    def split_text(self, text: str) -> list[str]:
        """递归分割文本"""
        final_chunks: list[str] = []
        # 获取当前分隔符
        separator = self._separators[-1]
        new_separators = []

        for i, _s in enumerate(self._separators):
            if _s == "":
                separator = _s
                break
            if _s in text:
                separator = _s
                new_separators = self._separators[i + 1:]
                break

        # 根据分隔符拆分
        if separator:
            splits = text.split(separator)
        else:
            splits = list(text)

        # 合并小块、拆分大块
        good_splits = []
        _separator = separator
        for s in splits:
            if self._length_function(s) < self._chunk_size:
                good_splits.append(s)
            else:
                if good_splits:
                    merged = _separator.join(good_splits)
                    final_chunks.extend(self._merge_splits(merged, _separator))
                    good_splits = []
                if not new_separators:
                    final_chunks.append(s)
                else:
                    other_info = RecursiveCharacterSplitter(
                        chunk_size=self._chunk_size,
                        chunk_overlap=self._chunk_overlap,
                        separators=new_separators
                    )
                    final_chunks.extend(other_info.split_text(s))

        if good_splits:
            merged = _separator.join(good_splits)
            final_chunks.extend(self._merge_splits(merged, _separator))

        return final_chunks

    def _merge_splits(self, text: str, separator: str) -> list[str]:
        """合并小的文本块"""
        splits = text.split(separator) if separator else [text]
        chunks = []
        current_chunk: list[str] = []
        current_length = 0

        for piece in splits:
            piece_len = self._length_function(piece)
            if current_length + piece_len > self._chunk_size and current_chunk:
                chunk_text = separator.join(current_chunk)
                if chunk_text.strip():
                    chunks.append(chunk_text)
                # 保留overlap部分
                while current_length > self._chunk_overlap and current_chunk:
                    removed = current_chunk.pop(0)
                    current_length -= self._length_function(removed)

            current_chunk.append(piece)
            current_length += piece_len

        if current_chunk:
            chunk_text = separator.join(current_chunk)
            if chunk_text.strip():
                chunks.append(chunk_text)

        return chunks


class SemanticChunker:
    """语义分块器 - 基于语义相似度的智能分块"""

    def __init__(self, embedding_model, threshold_percentile: int = 95):
        self._embedding_model = embedding_model
        self._threshold_percentile = threshold_percentile

    def split_text(self, text: str, sentence_split_regex: str = r'[。！？.!?]') -> list[str]:
        """基于语义相似度的分块"""
        import numpy as np

        # 1. 按句子拆分
        sentences = re.split(sentence_split_regex, text)
        sentences = [s.strip() for s in sentences if s.strip()]

        if len(sentences) <= 1:
            return [text]

        # 2. 计算每对相邻句子的嵌入相似度
        embeddings = self._embedding_model.embed_documents(sentences)
        similarities = []
        for i in range(len(embeddings) - 1):
            sim = np.dot(embeddings[i], embeddings[i + 1]) / (
                np.linalg.norm(embeddings[i]) * np.linalg.norm(embeddings[i + 1])
            )
            similarities.append(sim)

        # 3. 找到语义断点（相似度低于阈值的位置）
        threshold = np.percentile(similarities, 100 - self._threshold_percentile)
        breakpoints = [i + 1 for i, sim in enumerate(similarities) if sim < threshold]

        # 4. 根据断点分块
        chunks = []
        start = 0
        for bp in breakpoints:
            chunk = "。".join(sentences[start:bp]) + "。"
            if chunk.strip():
                chunks.append(chunk)
            start = bp

        # 处理最后一段
        if start < len(sentences):
            chunk = "。".join(sentences[start:]) + "。"
            if chunk.strip():
                chunks.append(chunk)

        return chunks
```

**第三阶段：向量化与存储**

```python
"""向量化与存储模块"""
from dataclasses import dataclass
from typing import Optional
import hashlib
import time


@dataclass
class VectorRecord:
    """向量记录"""
    id: str
    vector: list[float]
    content: str
    metadata: dict


class EmbeddingPipeline:
    """向量化流水线"""

    def __init__(self, embedding_model, vector_store, batch_size: int = 32):
        self._embedding_model = embedding_model
        self._vector_store = vector_store
        self._batch_size = batch_size

    async def process_chunks(
        self,
        chunks: list[str],
        metadata: dict
    ) -> list[str]:
        """批量处理分块并存储向量"""
        record_ids = []

        # 分批向量化
        for i in range(0, len(chunks), self._batch_size):
            batch = chunks[i:i + self._batch_size]
            # 调用Embedding模型生成向量
            vectors = await self._embedding_model.embed_documents(batch)

            # 构建向量记录
            records = []
            for j, (chunk, vector) in enumerate(zip(batch, vectors)):
                record_id = self._generate_id(chunk, metadata)
                record = VectorRecord(
                    id=record_id,
                    vector=vector,
                    content=chunk,
                    metadata={
                        **metadata,
                        "chunk_index": i + j,
                        "indexed_at": time.time()
                    }
                )
                records.append(record)
                record_ids.append(record_id)

            # 批量写入向量数据库
            await self._vector_store.upsert(records)

        return record_ids

    def _generate_id(self, content: str, metadata: dict) -> str:
        """基于内容哈希生成唯一ID"""
        raw = f"{metadata.get('file_name', '')}:{content}"
        return hashlib.sha256(raw.encode()).hexdigest()[:16]
```

### 四、检索优化

检索优化是RAG系统效果提升的关键。我们采用"混合检索 + Rerank + 查询改写"的三重优化策略。

**混合检索架构**：

```
用户查询
    │
    ▼
┌───────────────────┐
│   查询改写模块     │  ← 基于LLM的查询扩展和改写
│  (Query Rewrite)  │
└───────┬───────────┘
        │
   ┌────┴────┐
   ▼         ▼
┌───────┐ ┌────────┐
│向量检索│ │关键词检索│   ← 并行执行
│(Milvus)│ │  (ES)  │
└───┬───┘ └───┬────┘
    │         │
    └────┬────┘
         ▼
   ┌───────────┐
   │ 分数融合   │   ← RRF (Reciprocal Rank Fusion)
   │  (RRF)    │
   └─────┬─────┘
         ▼
   ┌───────────┐
   │ Reranker  │   ← Cross-Encoder 精排
   │ 精排重排   │
   └─────┬─────┘
         ▼
   ┌───────────┐
   │ Top-K 结果 │   → 送入LLM生成答案
   └───────────┘
```

```python
"""混合检索引擎"""
from dataclasses import dataclass
from typing import Optional
import numpy as np


@dataclass
class RetrievalResult:
    """检索结果"""
    content: str
    score: float
    source: str  # "vector" | "keyword" | "hybrid"
    metadata: dict


class HybridRetriever:
    """混合检索引擎 - 融合向量检索与关键词检索"""

    def __init__(
        self,
        vector_store,
        keyword_store,
        reranker,
        query_rewriter,
        top_k: int = 10,
        rerank_top_k: int = 5,
        rrf_k: int = 60
    ):
        self._vector_store = vector_store
        self._keyword_store = keyword_store
        self._reranker = reranker
        self._query_rewriter = query_rewriter
        self._top_k = top_k
        self._rerank_top_k = rerank_top_k
        self._rrf_k = rrf_k

    async def retrieve(
        self,
        query: str,
        collection: str = "default",
        filters: Optional[dict] = None
    ) -> list[RetrievalResult]:
        """执行混合检索"""
        # 1. 查询改写 - 扩展查询意图
        rewritten_queries = await self._query_rewriter.rewrite(query)

        # 2. 并行执行向量检索和关键词检索
        all_results: list[RetrievalResult] = []

        for rq in rewritten_queries:
            # 向量检索
            vector_results = await self._vector_store.search(
                query=rq,
                collection=collection,
                top_k=self._top_k,
                filters=filters
            )
            # 关键词检索
            keyword_results = await self._keyword_store.search(
                query=rq,
                collection=collection,
                top_k=self._top_k,
                filters=filters
            )
            all_results.extend(vector_results)
            all_results.extend(keyword_results)

        # 3. RRF分数融合
        fused_results = self._rrf_fusion(all_results)

        # 4. Reranker精排
        if self._reranker and len(fused_results) > self._rerank_top_k:
            fused_results = await self._reranker.rerank(
                query=query,
                documents=fused_results,
                top_k=self._rerank_top_k
            )

        return fused_results[:self._rerank_top_k]

    def _rrf_fusion(
        self,
        results: list[RetrievalResult]
    ) -> list[RetrievalResult]:
        """Reciprocal Rank Fusion 分数融合"""
        # 按内容去重并计算RRF分数
        content_scores: dict[str, float] = {}
        content_map: dict[str, RetrievalResult] = {}

        # 分别计算向量检索和关键词检索的排名
        vector_results = [r for r in results if r.source == "vector"]
        keyword_results = [r for r in results if r.source == "keyword"]

        for rank, result in enumerate(vector_results):
            key = result.content[:100]  # 用前100字符作为去重key
            rrf_score = 1.0 / (self._rrf_k + rank + 1)
            content_scores[key] = content_scores.get(key, 0) + rrf_score
            content_map[key] = result

        for rank, result in enumerate(keyword_results):
            key = result.content[:100]
            rrf_score = 1.0 / (self._rrf_k + rank + 1)
            content_scores[key] = content_scores.get(key, 0) + rrf_score
            content_map[key] = result

        # 按融合分数排序
        sorted_keys = sorted(content_scores.keys(), key=lambda k: content_scores[k], reverse=True)

        return [
            RetrievalResult(
                content=content_map[key].content,
                score=content_scores[key],
                source="hybrid",
                metadata=content_map[key].metadata
            )
            for key in sorted_keys
        ]


class QueryRewriter:
    """查询改写器 - 基于LLM的查询扩展"""

    SYSTEM_PROMPT = """你是一个查询改写专家。给定用户的原始查询，你需要生成2-3个不同角度的改写查询，
用于提高文档检索的召回率。改写查询应该：
1. 保持原始意图不变
2. 使用不同的关键词和表达方式
3. 考虑同义词替换和上下文扩展
4. 输出JSON数组格式"""

    def __init__(self, llm):
        self._llm = llm

    async def rewrite(self, query: str) -> list[str]:
        """改写查询"""
        response = await self._llm.ainvoke([
            {"role": "system", "content": self.SYSTEM_PROMPT},
            {"role": "user", "content": f"原始查询: {query}"}
        ])
        # 解析LLM返回的JSON数组
        import json
        try:
            rewritten = json.loads(response.content)
            return [query] + rewritten  # 保留原始查询
        except json.JSONDecodeError:
            return [query]
```

### 六、效果评估与持续优化

RAG系统的效果评估需要从多个维度进行量化：

```python
"""RAG效果评估框架"""
from dataclasses import dataclass
from typing import Optional


@dataclass
class EvaluationMetrics:
    """评估指标"""
    # 检索质量指标
    recall_at_k: float       # Top-K 召回率
    precision_at_k: float    # Top-K 精确率
    mrr: float               # Mean Reciprocal Rank
    ndcg_at_k: float         # Normalized Discounted Cumulative Gain

    # 生成质量指标
    faithfulness: float      # 忠实度 - 答案是否基于检索内容
    answer_relevancy: float  # 答案相关性
    context_precision: float # 上下文精确度
    context_recall: float    # 上下文召回率

    # 端到端指标
    accuracy: float          # 端到端准确率
    latency_p50: float       # P50 延迟
    latency_p99: float       # P99 延迟


class RAGEvaluator:
    """RAG评估器"""

    def __init__(self, llm, embedding_model):
        self._llm = llm
        self._embedding_model = embedding_model

    async def evaluate(
        self,
        test_cases: list[dict],
        rag_pipeline
    ) -> EvaluationMetrics:
        """执行完整评估"""
        results = []

        for case in test_cases:
            query = case["query"]
            ground_truth = case["ground_truth"]

            # 执行RAG流程
            answer, contexts = await rag_pipeline.query(query)

            # 计算各项指标
            result = {
                "query": query,
                "answer": answer,
                "ground_truth": ground_truth,
                "contexts": contexts,
                "faithfulness": await self._eval_faithfulness(answer, contexts),
                "relevancy": await self._eval_relevancy(query, answer),
                "context_precision": await self._eval_context_precision(query, contexts),
                "context_recall": await self._eval_context_recall(ground_truth, contexts)
            }
            results.append(result)

        return self._aggregate_metrics(results)

    async def _eval_faithfulness(self, answer: str, contexts: list[str]) -> float:
        """评估答案忠实度 - 答案中的每个声明是否都能在上下文中找到支持"""
        prompt = f"""请评估以下答案是否忠实于给定的上下文。
答案: {answer}
上下文: {' '.join(contexts)}

对答案中的每个声明，判断是否被上下文支持。
返回0-1之间的分数，1表示完全忠实。只返回数字。"""
        response = await self._llm.ainvoke([{"role": "user", "content": prompt}])
        try:
            return float(response.content.strip())
        except ValueError:
            return 0.0

    def _aggregate_metrics(self, results: list[dict]) -> EvaluationMetrics:
        """聚合评估指标"""
        n = len(results)
        return EvaluationMetrics(
            recall_at_k=sum(r.get("context_recall", 0) for r in results) / n,
            precision_at_k=sum(r.get("context_precision", 0) for r in results) / n,
            mrr=0.0,  # 需要排序信息计算
            ndcg_at_k=0.0,
            faithfulness=sum(r["faithfulness"] for r in results) / n,
            answer_relevancy=sum(r["relevancy"] for r in results) / n,
            context_precision=sum(r["context_precision"] for r in results) / n,
            context_recall=sum(r.get("context_recall", 0) for r in results) / n,
            accuracy=0.0,
            latency_p50=0.0,
            latency_p99=0.0
        )
```

## 深度解析

RAG系统的本质是一个**信息检索+生成**的两阶段系统。面试官考察的核心点不仅在于你是否了解各个组件，更在于你是否理解**每个设计决策背后的权衡（Trade-off）**。

**关键设计决策解析**：

1. **分块大小选择**：512 token 是经验值，太小会丢失上下文，太大会引入噪声。实际项目中需要通过A/B测试确定最优值。
2. **混合检索的必要性**：纯向量检索在精确匹配（如产品编号、法条编号）场景下表现不佳，需要关键词检索补充。
3. **Reranker的价值**：Bi-Encoder（向量检索）的精度不如Cross-Encoder（Reranker），但Cross-Encoder计算成本高，因此只对Top-K结果做精排。
4. **查询改写的边界**：过度改写会偏离原始意图，需要控制改写幅度和数量。

## 🧠 记忆口诀

**"解分向存，检索四步走"**

- **解**析：多格式文档统一解析
- **分**块：语义分块保完整
- **向**量化：Embedding转高维
- **存**储：向量库+ES双写

检索四步：**改**写→**混**合→**融**合→**精**排（"改混融精"）

## 🏠 生活类比

RAG系统就像一个**企业图书馆管理系统**：
- **文档解析**相当于图书管理员把各种格式的资料（报纸、杂志、书籍）都整理成统一格式的卡片
- **分块**相当于把每本书拆成章节卡片，每张卡片内容适中，不会太长也不会太短
- **向量化**相当于给每张卡片贴上"主题标签"，方便按主题查找
- **混合检索**相当于你不仅按主题标签找（向量），还按关键词目录找（ES），两份结果取并集
- **Reranker**相当于图书管理员在你找到的候选书籍中，仔细阅读后帮你排出最相关的前几本

## 🎯 面试追问

**追问1：分块策略如何选择？不同场景下有什么差异？**
> 回答要点：分块策略需要根据文档类型和查询模式选择。对于结构化文档（法律、合同），按段落/条款分块；对于叙述性文档（技术文档），按语义分块；对于表格密集文档，表格需要作为独立块处理。关键指标是块内信息完整性和块间边界合理性。

**追问2：如何处理多模态文档（包含图片和表格的PDF）？**
> 回答要点：使用多模态模型（如GPT-4V）对图片进行描述，表格提取后转为Markdown格式，图片描述和表格内容作为独立分块存储。需要维护原始位置映射关系，支持溯源。

**追问3：向量数据库选型考虑哪些因素？Milvus vs Qdrant vs Pinecone 怎么选？**
> 回答要点：考虑数据规模（Milvus适合亿级）、部署方式（Pinecone全托管 vs Milvus自部署）、过滤能力（Qdrant的payload过滤更灵活）、成本（开源 vs 商业）。国内企业倾向Milvus（国产+开源），海外SaaS倾向Pinecone。

**追问4：如何解决RAG系统中的"幻觉"问题？**
> 回答要点：三道防线——①检索阶段确保高召回（混合检索+查询改写）；②生成阶段使用严格的Prompt约束（"只基于提供的上下文回答"）；③后处理阶段做faithfulness检测，低置信度答案标注"不确定"。

## 🚀 AI应用扩展

- **多模态RAG**：支持图片、音视频内容的检索与生成
- **GraphRAG**：结合知识图谱的RAG，提升复杂关系推理能力
- **Agentic RAG**：Agent自主决定检索策略、工具调用顺序
- **Self-RAG**：模型自主判断是否需要检索、检索结果是否相关

## ⚠️ 容易踩坑

1. **分块时丢失表格/图片**：PDF解析时表格被拆散，图片被忽略。解决方案：表格整体作为一个分块，图片用多模态模型生成描述。
2. **向量模型与查询语言不匹配**：用英文Embedding模型处理中文文档，效果极差。解决方案：选择支持目标语言的模型（如bge系列）。
3. **忽略文档更新增量索引**：每次文档更新都全量重建索引，成本极高。解决方案：基于内容哈希做增量更新，只处理变更的分块。
4. **Top-K参数设置不当**：K太小漏检，K太大引入噪声。建议K=5~10，配合Reranker精排。
5. **生产环境不做查询缓存**：相同查询每次都走完整检索流程，浪费资源。建议对热点查询做Redis缓存，设置合理的TTL。

### 一、多轮对话架构

智能客服Agent的核心是维护一个**有状态的对话管理器**，能够在多轮交互中准确理解用户意图、维护上下文、调用工具并生成连贯回复。

**架构设计原则**：
- **对话状态机**：明确定义对话阶段（问候→意图识别→信息收集→处理→确认→结束）
- **上下文窗口管理**：滑动窗口 + 摘要压缩，避免token溢出
- **多Agent协作**：不同子Agent处理不同业务场景（售前、售后、投诉）

```python
"""智能客服Agent核心架构"""
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional, Any
import time
import json


class ConversationState(Enum):
    """对话状态枚举"""
    GREETING = "greeting"
    INTENT_DETECTION = "intent_detection"
    INFO_GATHERING = "info_gathering"
    PROCESSING = "processing"
    CONFIRMING = "confirming"
    TRANSFERRING = "transferring"  # 转人工
    CLOSING = "closing"


class Intent(Enum):
    """意图枚举"""
    PRODUCT_INQUIRY = "product_inquiry"
    ORDER_STATUS = "order_status"
    RETURN_REFUND = "return_refund"
    COMPLAINT = "complaint"
    TECHNICAL_SUPPORT = "technical_support"
    GENERAL_INQUIRY = "general_inquiry"
    HUMAN_TRANSFER = "human_transfer"
    UNKNOWN = "unknown"


@dataclass
class ConversationContext:
    """对话上下文"""
    session_id: str
    user_id: str
    state: ConversationState = ConversationState.GREETING
    intent: Optional[Intent] = None
    entities: dict = field(default_factory=dict)
    history: list[dict] = field(default_factory=list)
    collected_slots: dict = field(default_factory=dict)
    required_slots: list[str] = field(default_factory=list)
    satisfaction_scores: list[float] = field(default_factory=list)
    created_at: float = field(default_factory=time.time)
    turn_count: int = 0


class CustomerServiceAgent:
    """智能客服Agent"""

    def __init__(self, llm, intent_classifier, knowledge_base, tools):
        self._llm = llm
        self._intent_classifier = intent_classifier
        self._knowledge_base = knowledge_base
        self._tools = tools

    async def handle_message(
        self,
        message: str,
        context: ConversationContext
    ) -> tuple[str, ConversationContext]:
        """处理用户消息"""
        context.turn_count += 1
        context.history.append({"role": "user", "content": message})

        # 根据当前状态路由处理
        match context.state:
            case ConversationState.GREETING:
                response = await self._handle_greeting(message, context)
            case ConversationState.INTENT_DETECTION:
                response = await self._handle_intent_detection(message, context)
            case ConversationState.INFO_GATHERING:
                response = await self._handle_info_gathering(message, context)
            case ConversationState.PROCESSING:
                response = await self._handle_processing(message, context)
            case ConversationState.CONFIRMING:
                response = await self._handle_confirming(message, context)
            case ConversationState.TRANSFERRING:
                response = await self._handle_transfer(message, context)
            case _:
                response = "抱歉，我暂时无法处理您的请求，正在为您转接人工客服。"
                context.state = ConversationState.TRANSFERRING

        context.history.append({"role": "assistant", "content": response})
        return response, context

    async def _handle_intent_detection(
        self,
        message: str,
        context: ConversationContext
    ) -> str:
        """意图识别处理"""
        # 使用LLM进行意图分类
        intent_result = await self._intent_classifier.classify(
            message=message,
            history=context.history[-6:]  # 最近3轮对话
        )
        context.intent = intent_result.intent
        context.entities.update(intent_result.entities)

        # 根据意图配置所需信息槽位
        slot_config = self._get_required_slots(context.intent)
        context.required_slots = slot_config
        context.state = ConversationState.INFO_GATHERING

        # 检查是否有足够信息直接处理
        if self._all_slots_filled(context):
            context.state = ConversationState.PROCESSING
            return await self._handle_processing(message, context)

        # 引导用户补充信息
        missing_slot = self._get_next_missing_slot(context)
        return self._generate_slot_prompt(context.intent, missing_slot)

    async def _handle_processing(
        self,
        message: str,
        context: ConversationContext
    ) -> str:
        """业务处理"""
        # 1. 从知识库检索相关信息
        knowledge_results = await self._knowledge_base.search(
            query=message,
            filters={"intent": context.intent.value}
        )

        # 2. 根据意图调用相应工具
        tool_result = None
        match context.intent:
            case Intent.ORDER_STATUS:
                tool_result = await self._tools.call(
                    "query_order_status",
                    order_id=context.collected_slots.get("order_id")
                )
            case Intent.RETURN_REFUND:
                tool_result = await self._tools.call(
                    "create_return_request",
                    **context.collected_slots
                )
            case Intent.TECHNICAL_SUPPORT:
                tool_result = await self._tools.call(
                    "search_troubleshooting",
                    product=context.collected_slots.get("product"),
                    issue=context.collected_slots.get("issue_description")
                )

        # 3. 生成回复
        response = await self._generate_response(
            message=message,
            context=context,
            knowledge=knowledge_results,
            tool_result=tool_result
        )

        context.state = ConversationState.CONFIRMING
        return response

    def _get_required_slots(self, intent: Intent) -> list[str]:
        """获取意图对应的必需信息槽位"""
        slot_mapping = {
            Intent.ORDER_STATUS: ["order_id"],
            Intent.RETURN_REFUND: ["order_id", "reason", "product_name"],
            Intent.COMPLAINT: ["issue_description", "order_id"],
            Intent.TECHNICAL_SUPPORT: ["product", "issue_description"],
            Intent.PRODUCT_INQUIRY: ["product_name"],
        }
        return slot_mapping.get(intent, [])
```

### 二、意图识别与实体提取

"""意图识别与实体提取模块"""

@dataclass
class IntentResult:
    """意图识别结果"""
    intent: Intent

class LLMIntentClassifier:

    "confidence": 0.95,
    "entities": {

- product_inquiry: 产品咨询

        self,

        for msg in history[-4:]:
            messages.append(msg)
        messages.append({"role": "user", "content": f"用户消息: {message}"})

        try:
            result = json.loads(response.content)
            return IntentResult(
                intent=Intent(result.get("intent", "unknown")),
                confidence=result.get("confidence", 0.0),
                entities=result.get("entities", {}),

            )
        except (json.JSONDecodeError, ValueError):

                intent=Intent.UNKNOWN,
                confidence=0.0,
                entities={},

```python
"""知识库检索与FAQ匹配"""
from dataclasses import dataclass
from typing import Optional


@dataclass
class FAQMatch:
    """FAQ匹配结果"""
    question: str
    answer: str
    score: float
    category: str


class HybridFAQMatcher:
    """混合FAQ匹配器 - 结合语义匹配和精确匹配"""

    def __init__(self, embedding_model, vector_store, faq_index):
        self._embedding_model = embedding_model
        self._vector_store = vector_store
        self._faq_index = faq_index  # Elasticsearch FAQ索引

    async def match(
        self,
        query: str,
        top_k: int = 3,
        threshold: float = 0.75
    ) -> list[FAQMatch]:
        """匹配FAQ"""
        # 1. 精确匹配（关键词 + 编辑距离）
        exact_matches = await self._exact_match(query, top_k)

        # 2. 语义匹配（向量相似度）
        semantic_matches = await self._semantic_match(query, top_k)

        # 3. 合并结果，去重排序
        all_matches = self._merge_results(exact_matches, semantic_matches)

        # 4. 过滤低置信度结果
        return [m for m in all_matches if m.score >= threshold][:top_k]

    async def _semantic_match(self, query: str, top_k: int) -> list[FAQMatch]:
        """语义匹配"""
        query_vector = await self._embedding_model.embed_query(query)
        results = await self._vector_store.search(
            vector=query_vector,
            collection="faq",
            top_k=top_k
        )
        return [
            FAQMatch(
                question=r.metadata["question"],
                answer=r.metadata["answer"],
                score=r.score,
                category=r.metadata.get("category", "")
            )
            for r in results
        ]

    async def _exact_match(self, query: str, top_k: int) -> list[FAQMatch]:
        """精确匹配 - 使用ES的match查询"""
        results = await self._faq_index.search(
            query=query,
            fields=["question", "keywords"],
            top_k=top_k
        )
        return [
            FAQMatch(
                question=r["question"],
                answer=r["answer"],
                score=r["_score"],
                category=r.get("category", "")
            )
            for r in results
        ]

    def _merge_results(
        self,
        exact: list[FAQMatch],
        semantic: list[FAQMatch]
    ) -> list[FAQMatch]:
        """合并并去重"""
        seen_questions = set()
        merged = []

        # 精确匹配优先
        for m in exact:
            if m.question not in seen_questions:
                seen_questions.add(m.question)
                merged.append(m)

        for m in semantic:
            if m.question not in seen_questions:
                seen_questions.add(m.question)
                merged.append(m)

        return sorted(merged, key=lambda x: x.score, reverse=True)
```

class HumanCollaborationManager:

    def __init__(self, agent, queue_service, notification_service):
        self._agent = agent
        self._queue = queue_service
        self._notifier = notification_service

    async def evaluate_transfer_need(
        self,
        context: ConversationContext
    ) -> tuple[bool, Optional[TransferReason]]:
        """评估是否需要转人工"""

        if context.intent == Intent.HUMAN_TRANSFER:
            return True, TransferReason.USER_REQUEST

            return True, TransferReason.LOW_CONFIDENCE

            return True, TransferReason.COMPLAINT_ESCALATION

        # 规则4：LLM评估复杂度
        complexity = await self._assess_complexity(context)
        if complexity > 0.8:
            return True, TransferReason.COMPLEX_ISSUE

        return False, None

        self,
        context: ConversationContext,
        reason: TransferReason
    ) -> str:
        """转接人工客服"""
        # 1. 生成对话摘要

            reason=reason,
            conversation_summary=summary,

            attempted_actions=self._extract_actions(context.history),

"""满意度评估模块"""
from dataclasses import dataclass


@dataclass
class SatisfactionMetrics:
    """满意度指标"""
    csat_score: float          # 客户满意度 (1-5)
    resolution_rate: float     # 解决率
    avg_turns: float           # 平均对话轮数
    transfer_rate: float       # 转人工率
    response_accuracy: float   # 回答准确率


class SatisfactionEvaluator:
    """满意度评估器"""

    def __init__(self, llm):
        self._llm = llm

    async def evaluate_conversation(
        self,
        context: ConversationContext
    ) -> SatisfactionMetrics:
        """评估单次对话满意度"""
        # 1. 基于LLM的对话质量评估
        quality_score = await self._evaluate_quality(context)

        # 2. 用户行为信号分析
        behavior_score = self._analyze_behavior(context)

        # 3. 综合评分
        final_score = quality_score * 0.6 + behavior_score * 0.4

        return SatisfactionMetrics(
            csat_score=final_score,
            resolution_rate=1.0 if context.state == ConversationState.CLOSING else 0.0,
            avg_turns=context.turn_count,
            transfer_rate=1.0 if context.state == ConversationState.TRANSFERRING else 0.0,
            response_accuracy=quality_score
        )

    async def _evaluate_quality(self, context: ConversationContext) -> float:
        """LLM评估对话质量"""
        conversation_text = "\n".join(
            f"{msg['role']}: {msg['content']}" for msg in context.history
        )
        prompt = f"""请评估以下客服对话的质量（1-5分）：

{conversation_text}

评估维度：
1. 回答准确性（是否正确回答了用户问题）
2. 响应相关性（回答是否切题）
3. 服务态度（语气是否友好专业）
4. 解决效率（是否快速定位问题）

只返回一个1-5的数字。"""
        response = await self._llm.ainvoke([{"role": "user", "content": prompt}])
        try:
            return float(response.content.strip())
        except ValueError:
            return 3.0

    def _analyze_behavior(self, context: ConversationContext) -> float:
        """分析用户行为信号"""
        score = 3.0  # 基准分

        # 正面信号
        positive_keywords = ["谢谢", "好的", "明白了", "解决了", "满意"]
        negative_keywords = ["没用", "不对", "转人工", "投诉", "垃圾"]

        last_messages = [m["content"] for m in context.history[-3:] if m["role"] == "user"]
        for msg in last_messages:
            for kw in positive_keywords:
                if kw in msg:
                    score += 0.3
            for kw in negative_keywords:
                if kw in msg:
                    score -= 0.5

        return max(1.0, min(5.0, score))
```

## 深度解析

智能客服Agent的复杂度在于**对话管理的不确定性**。与单次问答不同，多轮对话需要处理上下文丢失、用户意图转换、信息槽位收集等问题。面试时需要重点展示对**对话状态机设计**和**异常处理流程**的理解。

关键设计决策：
1. **状态机 vs 纯LLM**：状态机保证流程可控，LLM提供灵活性。实践中采用"状态机做骨架 + LLM做填充"的混合模式。
2. **意图分类精度**：需要处理模糊意图（如"我想退货"既是退换货也可能是投诉），通过多轮澄清提升准确率。
3. **转人工时机**：过早转人工降低自动化率，过晚转人工降低满意度。需要动态评估。

### 一、自然语言转SQL

SQL Agent的核心是将用户的自然语言查询准确转换为可执行的SQL语句。这需要深入理解数据库Schema、表关系、业务语义，并处理各种边界情况。

```python
"""SQL Agent核心引擎"""
from dataclasses import dataclass, field
from typing import Optional, Any
from enum import Enum
import re
import sqlparse


class QueryStatus(Enum):
    """查询状态"""
    SUCCESS = "success"
    SYNTAX_ERROR = "syntax_error"
    EXECUTION_ERROR = "execution_error"
    SECURITY_VIOLATION = "security_violation"
    TIMEOUT = "timeout"


@dataclass
class SQLResult:
    """SQL执行结果"""
    status: QueryStatus
    sql: str
    columns: list[str]
    rows: list[list[Any]]
    row_count: int
    execution_time_ms: float
    error_message: Optional[str] = None


@dataclass
class SchemaInfo:
    """数据库Schema信息"""
    database: str
    tables: dict[str, "TableInfo"]


@dataclass
class TableInfo:
    """表信息"""
    name: str
    comment: str
    columns: list["ColumnInfo"]
    primary_keys: list[str]
    foreign_keys: dict[str, str]  # column -> referenced_table.column
    sample_rows: list[dict] = field(default_factory=list)


@dataclass
class ColumnInfo:
    """列信息"""
    name: str
    type: str
    comment: str
    nullable: bool
    is_unique: bool


class TextToSQLAgent:
    """自然语言转SQL Agent"""

    SYSTEM_PROMPT = """你是一个资深数据库专家。请根据用户的自然语言问题，生成对应的SQL查询。

数据库Schema信息：
{schema}

规则：
1. 只生成SELECT查询，禁止INSERT/UPDATE/DELETE/DROP等修改操作
2. 使用标准SQL语法，兼容MySQL 8.0+
3. 对于模糊查询，使用LIKE和通配符
4. 涉及时间范围时，使用明确的日期格式
5. 结果集限制在1000行以内
6. 优先使用表注释和列注释理解业务含义
7. 对于聚合查询，确保GROUP BY和SELECT列一致

输出格式（纯JSON）：
{{
    "thinking": "分析思路",
    "sql": "生成的SQL语句",
    "explanation": "SQL的自然语言解释",
    "assumptions": ["做出的假设1", "假设2"]
}}"""

    def __init__(self, llm, db_connector, schema_manager, safety_checker):
        self._llm = llm
        self._db = db_connector
        self._schema_manager = schema_manager
        self._safety_checker = safety_checker

    async def query(self, natural_language: str, database: str = "default") -> SQLResult:
        """执行自然语言查询"""
        # 1. 获取相关表的Schema信息
        relevant_tables = await self._schema_manager.get_relevant_tables(
            natural_language, database
        )
        schema_text = self._format_schema(relevant_tables)

        # 2. LLM生成SQL
        sql_response = await self._generate_sql(natural_language, schema_text)
        sql = sql_response["sql"]

        # 3. 安全检查
        safety_result = self._safety_checker.check(sql)
        if not safety_result.is_safe:
            return SQLResult(
                status=QueryStatus.SECURITY_VIOLATION,
                sql=sql,
                columns=[], rows=[], row_count=0,
                execution_time_ms=0,
                error_message=safety_result.reason
            )

        # 4. 执行SQL
        try:
            result = await self._db.execute(sql, timeout=30)
            return SQLResult(
                status=QueryStatus.SUCCESS,
                sql=sql,
                columns=result.columns,
                rows=result.rows[:1000],
                row_count=len(result.rows),
                execution_time_ms=result.execution_time_ms
            )
        except Exception as e:
            # 5. 错误修正重试
            return await self._retry_with_fix(natural_language, sql, str(e), schema_text)

    async def _retry_with_fix(
        self,
        question: str,
        failed_sql: str,
        error_message: str,
        schema_text: str
    ) -> SQLResult:
        """SQL执行失败后尝试修正"""
        fix_prompt = f"""SQL执行失败，请修正：

错误：{error_message}
Schema：{schema_text}

请分析错误原因并生成修正后的SQL。"""

## ⚡ 面试前5分钟冲刺

**Q: 如何包装一个RAG项目？**
30秒答：STAR法则——背景（企业知识分散，人工检索效率低）、任务（构建智能知识库，准确率>85%）、行动（混合检索+重排序+引用溯源）、结果（准确率88%，人工查询减少60%，月省15万人力）。

**Q: RAG系统最关键的优化点？**
30秒答：三处——分块策略（语义分块优于固定长度）、检索方式（向量+关键词+重排序混合检索优于纯向量）、Prompt设计（注入引用约束，强制LLM基于检索结果回答）。

**Q: SQL Agent怎么保证安全性？**
30秒答：四层防护——SQL白名单（只允许SELECT）、只读数据库账号、查询超时限制（30s）、结果行数限制（1000行）。危险操作（DROP/DELETE/UPDATE）直接拦截。

**Q: 智能客服Agent的多轮对话怎么实现？**
30秒答：对话状态机管理上下文，Slot Filling收集关键信息（订单号/问题类型），意图识别路由到对应技能，超时/无法识别触发兜底策略（转人工）。

**Q: 技术选型时怎么给面试官展示深度？**
30秒答：说清三个层次——1) 选了什么（结论）；2) 为什么选它（对比2-3个方案的优劣）；3) 有什么trade-off（不完美之处及应对方案）。

**Q: 项目成果怎么量化？**
30秒答：用具体数字——准确率（85%→92%）、延迟（3s→800ms）、成本（月费20万→8万）、效率（人工处理100单/天→系统处理2000单/天）、用户满意度（NPS 30→65）。

**Q: 面试中遇到不会的技术问题怎么办？**
30秒答：坦诚说"这块我了解不深"，然后说"但根据我的经验，我会这样思考..."，展示解决问题的思维过程比硬背答案更重要。

**Q: 如何展示"从0到1"的能力？**
30秒答：讲清楚完整链路——需求分析（和业务方对齐）→技术选型（对比方案）→架构设计（画图说话）→开发迭代（MVP→优化）→上线运维（监控+迭代），每个环节都有你主导的决策。
